# Writing Profile — Terminology

## Accepted Technical Terms (always in English, even in Hebrew sections)
- Artificial Intelligence, Machine Learning, Deep Learning, Neural Network
- Large Language Model (LLM), Transformer, Attention Mechanism
- API (Application Programming Interface), SDK, MCP (Model Context Protocol)
- LaTeX, LuaLaTeX, BibTeX, biber
- Precision, Recall, F1-Score, ROC curve, AUC
- Agent, Tool, Crew, Task, Pipeline

## Preferred Synonyms
| Avoid | Prefer |
|---|---|
| AI system | AI model / AI agent |
| Use | Utilize → use (shorter is better) |
| In order to | To |
| Due to the fact that | Because |
| It can be seen that | (delete — say the observation directly) |

## Citation Style — IEEE
- Inline: `\cite{author2024}` → appears as [1], [2], ...
- Multiple: `\cite{a2024, b2023}` → [1, 2]
- BibTeX key format: `firstauthorYEAR` (e.g., `topol2019`)
- Every factual claim must be followed by `\cite{}`
- "According to [1]..." is acceptable but not required

## Number Formatting
- Numbers > 999: use comma separator → 10,000 not 10000
- Percentages: always numeric → 85% not eighty-five percent
- Dates in text: ISO 8601 → 2024-03-15
- Dates in bibliography: YYYY-MM-DD format

## Hebrew Transliteration (when writing Hebrew terms in English text)
- Follow Academy of the Hebrew Language standards
- Example: בינה מלאכותית → bina melakutit (for academic glossary only)
- Technical terms stay in English: Machine Learning, not למידת מכונה (in English paragraphs)

## Terms to Avoid
- "leverage" → use "use" or "apply"
- "utilize" → use "use"
- "paradigm shift" → too vague, be specific
- "state of the art" → use the actual metric
- "it is worth noting that" → delete
- Any contractions: isn't, doesn't, can't → is not, does not, cannot
