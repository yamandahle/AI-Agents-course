# Article Writer — Multi-Agent AI System

A multi-agent AI pipeline that autonomously researches a topic, pre-generates publication-quality matplotlib charts, writes a 15-25 page bilingual (English + Hebrew) LuaLaTeX academic article in MDPI journal style, and iteratively reviews and edits it through 3 structured review cycles.

## Quick Start

```bash
# 1. Enter the project
cd HW3Version1

# 2. Copy and fill environment variables
cp .env-example .env
# Edit .env — set LLM_PROVIDER and API keys

# 3. Install dependencies
uv sync

# 4. Run the full pipeline (research + charts + write + review + compile)
uv run python src/main.py --mode full

# 5. Research only
uv run python src/main.py --mode research

# 6. Write from existing research.md
uv run python src/main.py --mode write
```

## How It Works

### Architecture

```
User
 │
 ├─► ResearcherAgent (CrewAI, HITL)
 │       │  Tools: deep_research, researcher_handler, citation_extractor, content_filter
 │       │  Search: Gemini 2.5 Flash → Perplexity fallback
 │       │  Output: data/research.md
 │       └─► Human approves each search batch before next batch
 │
 └─► WriterAgent (CrewAI, deterministic)
         │
         ├─ Phase 0: ChartGenerator — pre-generates 3 matplotlib PDFs (idempotent)
         ├─ Phase 1: ContextLoader loads guideline + research + profiles + PDF few-shots
         ├─ Phase 2: DraftGenerator → draft_v1.tex
         │             (LLM references pre-generated charts via \includegraphics)
         └─ Phase 3: ReviewLoop (3 iterations)
                  │
                  ├─► Reviewer (SEPARATE CONTEXT)
                  │       Sees: draft + guideline + research + profiles ONLY
                  │       Checks (in priority order):
                  │         1. Coverage — page count, missing sections, equation/figure/table
                  │         2. Structure — BiDi correctness, section order, titlepage
                  │         3. TikzLayout — spatial collisions: missing fill=white on labels,
                  │                         diagonal paths crossing boxes
                  │         4. Accuracy — factual errors vs research notes
                  │         5. Terminology / Characters
                  │       Output: ArticleReview Pydantic model
                  │         .comments: list[ReviewComment(profile, location, comment)]
                  │         .overall_score: float 0–10
                  │         .pass_fail: "PASS" | "FAIL"
                  │
                  └─► Editor (FULL CONTEXT)
                          Sees: draft + reviews + guideline + research + profiles + few-shots
                          Applies corrections in priority order:
                            1. Hebrew BiDi correctness (HIGHEST)
                            2. TikZ spatial collision fixes (orthogonal routing, fill=white)
                            3. Coverage violations (page count, missing elements)
                            4. Accuracy and Citation issues
                            5. Structure, Terminology, Characters
                          Output: draft_v{N+1}.tex

         Final: draft_final.tex → LatexSanitizer (13 fixes) → lualatex × 4 passes
              → article_final.pdf
```

### Agent Contexts (Strictly Enforced)

| Agent | Can See | Cannot See |
|---|---|---|
| **Researcher** | guideline, web search results, user feedback | draft, profiles, few-shots |
| **Writer (Draft + Edit)** | guideline, research, all 3 profiles, few-shot PDFs, chart paths | reviewer output during drafting |
| **Reviewer** | draft, guideline, research, profiles | few-shot examples, writer system prompt |
| **Editor** | draft, reviewer comments, guideline, research, profiles, few-shots | — (has full context) |

### Python-First Chart Pipeline

Before the LLM writes a single word, `chart_generator.py` pre-generates three publication-quality matplotlib PDFs:

| Chart | File | Content |
|---|---|---|
| Accuracy curve | `accuracy_curve.pdf` | Training vs validation accuracy over 30 epochs |
| Diagnostic comparison | `diagnostic_comparison.pdf` | AI vs human expert on 5 imaging tasks (bar chart) |
| Cost reduction | `cost_reduction.pdf` | Operational cost reduction across hospital departments |

The draft prompt tells the LLM these files exist and to reference them with `\includegraphics`. The sanitizer's `_fix_includegraphics` step skips replacement when the referenced PDF is found on disk — so charts appear as clean, properly anti-aliased images instead of hand-drawn pgfplots.

`generate_all()` is idempotent: existing PDFs are skipped unless `regenerate=True`.

### Tracing and Metrics

Every LLM call and tool invocation is logged:

- **`results/traces.jsonl`** — `{ts, step, provider, model, input_tokens, output_tokens, input, output, metadata}` per call
- **`results/metrics.jsonl`** — `{ts, step, model, latency_ms, input_tokens, output_tokens, total_tokens, cost_usd}` per call

Run `notebooks/results_analysis.ipynb` to visualise latency distribution, token usage over time, cumulative cost, cost by category, and model comparison.

### Eval Dataset Pipeline

```
1. ArticleExtractor  — reads MDPI PDFs (13–20 pages) via PyMuPDF
2. DatasetBuilder    — LLM labels each article PASS/FAIL + 3-sentence critique
                        splits: ~80% train / ~10% dev / ~10% test  (saved as JSONL)
3. ArticleJudge      — LLM judge predicts PASS/FAIL given guideline + research + article
4. JudgeLoop         — runs judge on dev split, computes F1, refines prompt, repeats
                        stops when F1 ≥ 0.80 or max_iterations (default 5) reached
5. Final validation  — runs winning judge on test split (held-out throughout)
```

## LaTeX Sanitizer Pipeline

Before every lualatex compilation, the 13-step deterministic sanitizer (`latex_sanitizer.py`) runs in order:

| Step | Fix | Description |
|---|---|---|
| n0 | `_fix_wrong_main_language` | Forces `\setmainlanguage{english}` and `\setotherlanguage{hebrew}`; injects `\setlength{\headheight}{14pt}` if missing |
| n1 | `_fix_malformed_envs` | 6 sub-patterns: `\end{env>` typos, `</env>` HTML tags, mixed `</env}`, tabularx/tabular mismatch, unclosed `\begin{axis}`, bare `&` in TikZ node labels |
| n2 | `_fix_missing_bidi_envs` | Injects missing `\begin{RTL}` / `\begin{LTR}` wrappers in Hebrew/Arabic sections |
| n3 | `_fix_includegraphics` | Replaces `\includegraphics` blocks with pgfplots/tikz — **skips when referenced PDF exists on disk** (lets pre-generated charts through) |
| n4 | `_fix_undefined_tikz_styles` | Replaces figures whose tikzpicture uses LLM-invented node styles before `\tikzset` |
| n5 | `_fix_cover_hebrew` | Wraps bare `{\hebrewfont ... hebrew-text}` on titlepage inside `\begin{hebrew}...\end{hebrew}` |
| n6 | `_fix_bidi_body_sections` | Enforces `\begin{RTL}` on body sections that contain Hebrew blocks |
| n7 | `_fix_cover_metadata` | Strips Hebrew label words from the titlepage `{rl}` metadata table (prevents BiDi mirroring) |
| n8 | `_fix_missing_ragged2e` | Normalises any `ragged2[a-z]+` typo to `ragged2e`, deduplicates, injects `\usepackage{ragged2e}` before tabularx |
| n9 | `_fix_table_alignment` | Body-only: converts `\begin{tabular}` → `\begin{tabularx}{\textwidth}` with `>{\RaggedRight\arraybackslash}X` columns |
| n10 | `_fix_arrow_labels` | Injects `fill=white, inner sep=2pt, font=\footnotesize` into every TikZ edge-label node missing `fill=` |
| n11 | `_fix_invalid_cite_keys` | Remaps any `\cite{key}` not in the 30-key curated set using 50+ keyword→key rules |
| n12 | `_fix_diagonal_paths` | Converts `-- ++(x,y)` diagonal coordinate waypoints (both x,y non-zero) to two-step orthogonal `-- ++(x,0) -- ++(0,y)` |

The sanitizer is body-aware: steps that could corrupt the titlepage `{rl}` metadata table only run on content after `\end{titlepage}`.

### 30-Key Citation Allow-List

`references.bib` contains exactly 30 curated entries — 11 `@article` with DOIs, 3 `@book`, 16 `@misc` with specific deep-link URLs. Every entry has: Author, Title, Journal/Institution, Year, and a specific (non-root-domain) URL.

The same 30 keys are enforced at three layers:

1. **Prompt** — `⚠ CITATION KEYS — HARD STOP` block in the writer system prompt lists all 30 keys with ✓/✗ examples
2. **Self-correction pass** — `_repair_cite_keys()` in `DraftGenerator` catches invalid keys immediately after LLM response and sends a targeted correction prompt before saving
3. **Sanitizer safety net** — `_fix_invalid_cite_keys()` remaps any surviving invalid keys using keyword rules before compilation

### TikZ Spatial Collision Prevention

Three-layer defence against arrow lines bleeding through text labels and diagonal paths crossing boxes:

1. **Writer prompt §3 TIKZ FLOWCHART RULES** — explicit rules with copy-paste examples:
   - Every `node[midway, ...]` label **must** carry `fill=white, inner sep=2pt` (white background masks the arrow line)
   - Feedback/return paths **must** use orthogonal routing (`|-`, `-|`, or `to[out,in]`) — never bare diagonal `--`
   - Return paths must be shifted ≥1.5 cm beyond the rightmost box border using `xshift`

2. **Reviewer TikzLayout check** (priority 3) — rejects figures where:
   - Any connection label node lacks `fill=white` or `fill=pagecolor`
   - Any `\draw` path uses bare `--` for a multi-hop route that could cross a box

3. **Sanitizer Fix n10 + n12**:
   - `_fix_arrow_labels` — adds `fill=white` to any naked edge-label node
   - `_fix_diagonal_paths` — converts `-- ++(x,y)` diagonal segments to `-- ++(x,0) -- ++(0,y)`

## Chart Self-Check Tool

A zero-API CLI tool for verifying **Structural Code Alignment** — that figure axis labels match the prose.

```bash
uv run python src/article_writer/tools/chart_checker.py
# Output → results/chart_check/chart_check.pdf
```

**What it does (no LLM calls):**
1. Extracts the first pgfplots `ybar` figure from `results/draft_final.tex`
2. Checks every axis tick label appears **verbatim** in the 3 paragraphs preceding the figure — prints PASS/FAIL per label
3. Extracts quantitative % facts from `data/research.md` (regex, no API)
4. Adds 3 known values from curated `@article` references (Esteva 91%, Rajpurkar 90.3%, Campanella 98.8%)
5. Compiles a 2-page standalone PDF: page 1 = draft figure + alignment table + prose context; page 2 = research facts bar chart + source sentences
