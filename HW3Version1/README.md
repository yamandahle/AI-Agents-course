# Article Writer — Multi-Agent AI System

A two-agent AI network that autonomously researches a topic, writes a 15-page LaTeX academic article in MDPI journal style, and iteratively reviews and edits it through 3–4 structured review cycles.

## Quick Start

```bash
# 1. Clone and enter the project
cd HW3Version1

# 2. Copy and fill environment variables
cp .env-example .env
# Edit .env — set LLM_PROVIDER and API keys

# 3. Install dependencies
uv sync

# 4. Run the full pipeline (research + write + review + compile)
uv run python src/main.py --mode full

# 5. Research only
uv run python src/main.py --mode research

# 6. Write from existing research.md
uv run python src/main.py --mode write
```

## How It Works

### Architecture

```
User
 │
 ├─► ResearcherAgent (CrewAI, HITL)
 │       │  Tools: deep_research, researcher_handler, citation_extractor, content_filter
 │       │  Search: Gemini 1.5 Pro → Perplexity fallback
 │       │  Output: data/research.md
 │       └─► Human approves each search batch before next batch
 │
 └─► WriterAgent (CrewAI, deterministic)
         │
         ├─ Phase 1: ContextLoader loads guideline + research + profiles + PDF few-shots
         ├─ Phase 2: DraftGenerator → draft_v1.tex
         └─ Phase 3: ReviewLoop (3–4 iterations)
                  │
                  ├─► Reviewer (SEPARATE CONTEXT)
                  │       Sees: draft + guideline + research + profiles ONLY
                  │       Output: review_v{N}.json  (ArticleReview Pydantic model)
                  │         ArticleReview.comments: list[ReviewComment]
                  │           ReviewComment.profile    — which constraint was violated
                  │           ReviewComment.location   — where in the article
                  │           ReviewComment.comment    — what is wrong (1-2 lines)
                  │         ArticleReview.overall_score — float 0–10
                  │         ArticleReview.pass_fail    — "PASS" | "FAIL"
                  │
                  └─► Editor (FULL CONTEXT)
                          Sees: draft + reviews + guideline + research + profiles + few-shots
                          Applies corrections in priority order:
                            1. Guideline violations (Coverage, Accuracy, Citation)
                            2. Research/accuracy violations
                            3. Profile violations (Structure, Terminology, Characters)
                          Output: draft_v{N+1}.tex

         Final: draft_final.tex → lualatex × 4 passes → article_final.pdf
```

### Agent Contexts (Strictly Enforced)

| Agent | Can See | Cannot See |
|---|---|---|
| **Researcher** | guideline, web search results, user feedback | draft, profiles, few-shots |
| **Writer (Draft + Edit)** | guideline, research, all 3 profiles, few-shot PDFs | reviewer output during drafting |
| **Reviewer** | draft, guideline, research, profiles | few-shot examples, writer system prompt |
| **Editor** | draft, reviewer comments, guideline, research, profiles, few-shots | — (has full context) |

### Tracing and Metrics

Every LLM call and tool invocation is logged:

- **`results/traces.jsonl`** — `{ts, step, provider, model, input_tokens, output_tokens, input, output, metadata}` per call
- **`results/metrics.jsonl`** — `{ts, step, model, latency_ms, input_tokens, output_tokens, total_tokens, cost_usd}` per call

### Eval Dataset Pipeline

```
1. ArticleExtractor  — reads MDPI PDFs (13–20 pages) via PyMuPDF
2. DatasetBuilder    — LLM labels each article PASS/FAIL + 3-sentence critique
                        splits: ~80% train / ~10% dev / ~10% test  (saved as JSONL)
3. ArticleJudge      — LLM judge predicts PASS/FAIL given guideline + research + article
4. JudgeLoop         — runs judge on dev split, computes F1, refines prompt, repeats
                        stops when F1 ≥ 0.80 or max_iterations (default 5) reached
5. Final validation  — runs winning judge on test split (held-out throughout)
```

## LaTeX Sanitizer Pipeline

Before every lualatex compilation, the 12-step deterministic sanitizer (`latex_sanitizer.py`) runs in order:

| Step | Fix |
|---|---|
| n0 — `_fix_malformed_envs` | 6 sub-patterns: `\end{env>` typos, `</env>` HTML tags, tabularx/tabular mismatch, unclosed `\begin{axis}`, bare `&` in TikZ node labels |
| n1 — `_fix_missing_ragged2e` | Normalises any `ragged2[a-z]+` typo to `ragged2e`, deduplicates, injects before tabularx |
| n2 — `_fix_table_alignment` | Body-only (skips titlepage): converts `\begin{tabular}` → `\begin{tabularx}{\textwidth}` with `>{\RaggedRight\arraybackslash}X` columns |
| n3 — `_fix_bidi_body_sections` | Injects `\begin{RTL}` / `\begin{LTR}` wrappers in Hebrew/Arabic sections |
| n4 — `_fix_arrow_labels` | Injects `fill=white, inner sep=2pt, font=\footnotesize` into every TikZ edge-label node missing `fill=` |
| n5 — `_fix_invalid_cite_keys` | Remaps any `\cite{key}` not in the 30-key curated set using 50+ keyword→key rules; deduplicates multi-key citations |
| n6–n11 | Additional document-structure and encoding fixes |

The sanitizer is body-aware: steps that could corrupt the titlepage `{rl}` metadata table only run on content after `\end{titlepage}`.

### 30-Key Citation Allow-List

`references.bib` contains exactly 30 curated entries — 11 `@article` with DOIs, 3 `@book`, 16 `@misc` with specific deep-link URLs. Every entry has: Author, Title, Journal/Institution, Year, and a specific (non-root-domain) URL.

The same 30 keys are enforced at three layers:

1. **Prompt** — `⚠ CITATION KEYS — HARD STOP` block at the top of the writer system prompt lists all 30 keys with ✓/✗ examples
2. **Self-correction pass** — `_repair_cite_keys()` in `DraftGenerator` catches invalid keys immediately after LLM response and sends a targeted correction prompt before saving
3. **Sanitizer safety net** — `_fix_invalid_cite_keys()` remaps any surviving invalid keys using keyword rules before compilation

### TikZ Arrow Label Clearance

Every `\draw[->] node[...]` edge label in TikZ diagrams automatically gets `fill=white, inner sep=2pt, font=\footnotesize` injected by the sanitizer to prevent text overlapping adjacent boxes. The hard rule is also stated in the writer prompt.

## Chart Self-Check Tool

A zero-API CLI tool for verifying **Structural Code Alignment** — that figure axis labels match the prose.

```bash
uv run python src/article_writer/tools/chart_checker.py
# Output → results/chart_check/chart_check.pdf
```

**What it does (no LLM calls):**
1. Extracts the first pgfplots `ybar` figure from `results/draft_final.tex`
2. Checks every axis tick label appears **verbatim** in the 3 paragraphs preceding the figure — prints PASS/FAIL per label
3. Extracts quantitative % facts from `data/research.md` (regex, no API)
4. Adds 3 known values from curated `@article` references (Esteva 91%, Rajpurkar 90.3%, Campanella 98.8%)
5. Compiles a 2-page standalone PDF: page 1 = draft figure + alignment table + prose context; page 2 = research facts bar chart + source sentences
